# Version 0.1.11

## Schema migration + settings store + unified node view

This round folds together several structural changes that had been building
up in `docs/TASK.md`:

1. the explicit `Event.category` discriminator (leftover tech debt from 0.1.7);
2. static financial metadata on `Entity` so the graph can size nodes by
   something other than event count, and so the SidePanel has real numbers
   to show;
3. a persisted client-side settings store with export/import;
4. collapsing News / Career into a single unified graph with per-entity Jobs
   tab living inside the SidePanel instead of a separate sidebar.

### Schema: `Event.category` + Entity financial fields

- **[prisma/schema.prisma](prisma/schema.prisma)** adds
  `category String?` on `Event` (values `"news"` / `"job"`; null tolerated
  so legacy rows stay visible in the news bucket) and five static fields on
  `Entity`: `stockTicker`, `marketCapUsd`, `employeeCount`, `freeCashFlow`,
  `foundedYear`. All optional — `/api/node` surfaces them as nulls if unset
  and the UI conditionally renders only the stats that have values.
- **[prisma/migrate_event_category.ts](prisma/migrate_event_category.ts)**
  backfills the new column via `$executeRawUnsafe` (three passes: single-agency
  `entity.type='agency'` → `job`, multi-entity → `news`, everything else →
  `news`). Ran clean: 5 job rows, 13 news rows.
- **[skills/news_crawl/update_news.ts](skills/news_crawl/update_news.ts)**,
  **[skills/job_crawl/update_jobs.ts](skills/job_crawl/update_jobs.ts)** and
  **[skills/news_crawl/seed_glasswing.ts](skills/news_crawl/seed_glasswing.ts)**
  all write `category` on event creation so the discriminator stays coherent
  going forward.
- **[skills/news_crawl/seed_financials.ts](skills/news_crawl/seed_financials.ts)**
  is a new idempotent script that upserts the static financial fields on the
  14 Glasswing seed entities (rough snapshots — real-time pulls belong in
  the pipeline, not the seed). Populates Anthropic / Palantir / AWS / Google /
  Microsoft / CrowdStrike / JPMorgan tickers + market cap + FCF + employees
  + founded-year so the node sizer and SidePanel stats have plausible inputs.
  Ran clean: 14 updated, 0 missing.

### API: `/api/graph` + `/api/node` reshape

- **[src/app/api/graph/route.ts](src/app/api/graph/route.ts)** now filters
  by `category` directly (`kind !== "all" ? { category: kind } : {}`) and
  exposes the raw size-factor fields (`eventCount`, `marketCapUsd`,
  `employeeCount`, `freeCashFlow`) on each node. The client picks which one
  drives radius via `computeNodeSize()` — the API stays agnostic.
- **[src/app/api/node/route.ts](src/app/api/node/route.ts)** pulls up to 30
  events per entity in one query, then splits them client-side into
  `recentEvents` (`category !== 'job'`) and `recentJobs` (`category === 'job'`),
  capping each at 10. It also returns the new financial fields on the entity
  payload so the SidePanel doesn't need a second round-trip.

### Client: settings store + unified graph + Jobs tab

- **[src/lib/useAppSettings.ts](src/lib/useAppSettings.ts)** is a new
  singleton-backed settings store using `useSyncExternalStore`. Types:
  `NodeSizeFactor = "event_count" | "market_cap" | "employee_count" |
  "free_cash_flow"` and `AppSettings = { theme, nodeSizeFactor }`. Persists
  to `localStorage["sheaf-settings-v1"]` on every mutation and applies the
  theme class to `document.documentElement` so Tailwind `dark:` variants
  activate without a context provider. Exposes `updateSettings`,
  `exportSettingsJson`, and `importSettingsJson` as module-level functions.
- **[src/app/page.tsx](src/app/page.tsx)** drops the News / Career tab row
  entirely (the tabs were redundant once the unified graph + per-entity Jobs
  tab shipped — task 7). Adds a small glass-panel "Sheaf" title top-left,
  pulls `settings.nodeSizeFactor` out of the store, and threads it into
  `GraphCanvas`. Default `timeFilter` stays at `9999` (All Time) from 0.1.10.
- **[src/components/CareerSidebar.tsx](src/components/CareerSidebar.tsx)**
  deleted — all its responsibilities now live inside the SidePanel Jobs tab.
- **[src/components/GraphCanvas.tsx](src/components/GraphCanvas.tsx)** accepts
  a `sizeFactor` prop and calls the new `computeNodeSize(factor, n)` helper.
  Financial fields are log-scaled against a pivot (`1e11` for market cap,
  `1e5` for employees, `1e10` for FCF) so billions and millions both render
  visibly — the client falls back to the event-count baseline when a field
  is null. The `SigmaContainer` `key` now includes `sizeFactor` so switching
  the setting remounts with a fresh layout worker.
- **[src/components/SidePanel.tsx](src/components/SidePanel.tsx)** node view
  gains two new sections:
  - **Key Stats grid** — renders `stockTicker`, `marketCapUsd`,
    `freeCashFlow`, `employeeCount`, `foundedYear` as glass tiles with the
    appropriate icon, only showing tiles whose value is set. FCF tile flips
    from emerald → red when negative.
  - **Events / Jobs tabs** — segmented control above the recent-events list.
    Each tab preserves the 0.1.10 "first click expands, second click opens
    source" interaction. Tab counts are shown in the label.
- **[src/components/SettingsPanel.tsx](src/components/SettingsPanel.tsx)**
  now reads/writes through `useAppSettings` instead of touching
  `document.documentElement` directly. Adds:
  - a **Node Size Factor** selector (2×2 card grid with hints);
  - an **Export JSON** button that downloads the current settings as
    `sheaf-settings-<date>.json` via a `Blob` + `URL.createObjectURL` (no
    server round-trip);
  - an **Import JSON** file input that pipes the file through
    `importSettingsJson` and reports success/failure inline.

### Task 1 deleted as moot

The "multi-file ingestion in the News tab via OpenClaw" item (deferred from
0.1.7) was removed from [docs/TASK.md](docs/TASK.md). After the 0.1.11 tab
consolidation there is no News tab to host an uploader, and the
`skills/news_crawl` CLI already covers bulk ingestion with the same dedup
protocol. If the feature comes back it will do so as a general "ingest
files" affordance rather than a tab-specific one.
