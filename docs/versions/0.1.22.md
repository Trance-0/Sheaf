# Version 0.1.22

## Deterministic graph layout + no single-vertex components

The graph used to start every session at random positions and let
ForceAtlas2 sort it out. Two problems:

1. **Chaotic layout.** High-impact nodes could end up anywhere, and
   reloads would rotate the whole graph into a different shape.
2. **Orphaned dots.** Filter windows that excluded the multi-entity
   events an entity participated in would leave that entity rendered
   as a lone vertex with no edges.

### Data audit (behind the fix)

A one-off `scripts/inspect_graph_data.ts` (since removed) queried the
live Neon DB:

- 136 total events: 104 with 2 entities, 32 with 1 entity.
- The 32 singletons are 29 jobs + 3 news.
- Every entity appears in at least one multi-entity event globally.

So "orphan nodes" are a filter-window artifact, not a data bug. The fix
is a client-side prune: keep only nodes that have at least one edge
after aggregation in the current window.

### Changes

- **[src/lib/client/graphData.ts](../../src/lib/client/graphData.ts)** —
  after building nodes + edges, compute the set of connected node ids
  and filter the node list to that set. No single-vertex components
  reach the renderer.
- **[src/lib/client/layout.ts](../../src/lib/client/layout.ts)** (NEW) —
  `computeRadialSeed(graph)` sorts nodes by degree (primary) and
  eventCount (tiebreaker), places the top-ranked node at the origin,
  and spirals the rest out in concentric rings with alternating angle
  offsets so adjacent rings don't align spokes.
  `relaxNoverlap(graph)` wraps `graphology-layout-noverlap` for the
  post-ForceAtlas2 pass that pushes overlapping nodes apart.
  `FORCE_ATLAS2_SETTINGS` centralises the FA2 tuning —
  `linLogMode: true`, `outboundAttractionDistribution: true`,
  `scalingRatio: 12`, `edgeWeightInfluence: 0` — picked to give roughly
  even edge lengths and a clear hub region.
- **[src/components/GraphCanvas.tsx](../../src/components/GraphCanvas.tsx)** —
  uses the radial seed instead of `(Math.random() - 0.5) * 1000`, runs
  FA2 for 4 seconds, then calls `relaxNoverlap` + `sigma.refresh()` so
  the final frame has no stacked nodes.

### New dependencies

- `graphology-layout-noverlap` — small plugin, main-thread relaxation.
- `graphology-layout` — installed as a common companion; not directly
  imported yet but kept for future circular / random seed utilities.
