# Version 0.1.8

## Project Glasswing Deep Seed

Initial data population for Anthropic's Project Glasswing and its surrounding ecosystem of defense, intelligence, national-lab, and cybersecurity partners. All events were sourced from primary announcements (Anthropic blog, GSA.gov, Energy.gov, Palantir investor relations) or reputable secondary coverage (CNBC, TechCrunch, Breaking Defense, Axios).

### Seed artifacts

- [skills/news_crawl/glasswing_seed.json](skills/news_crawl/glasswing_seed.json) — 14 entities + 25 source events with real article URLs and ISO dates.
- [skills/news_crawl/seed_glasswing.ts](skills/news_crawl/seed_glasswing.ts) — one-shot TypeScript loader that upserts the entities (overwriting `type`, `homepage`, `jobPortal` to repair earlier placeholder rows) and runs each event through the same entity-pair keyword dedup as `update_news.ts`.

### Seed run result

- 14 entities upserted — Anthropic, Palantir, AWS, Google, Microsoft, CrowdStrike, Carahsoft, Linux Foundation, JPMorgan Chase, DoD, DOE, NNSA, GSA, Lawrence Livermore National Laboratory. Types range across `company`, `agency`, `lab`.
- 15 new events created, 10 article references attached to existing events via the dedup protocol.
- Verified via API:
  - `/api/graph?days=9999&kind=news` → 14 nodes, 13 edges (Anthropic-DoD and Anthropic-AWS each carry 2 events).
  - `/api/edge?source=anthropic&target=dod` → 2 events, 6 articles combined.
  - `/api/node?id=anthropic` → type `company`, homepage + job portal populated, 10 recent events.

### Entity type corrections

The previous seed (0.1.2) classified Anthropic/Google/Microsoft as `agency`. The loader now corrects those to `company`.

### Known dedup edge case

The keyword-match dedup is aggressive: for example "Anthropic loses appeals bid to block Pentagon blacklisting" and "CDAO splits $600M..." both link Anthropic↔DoD and share overlapping keywords, so the former was attached as an article on the latter. For a tighter seed we would also compare publication dates, but that is intentional tech debt — the rule is consistent with `update_news.ts` and easy to revisit once the `Event.category` migration lands.
