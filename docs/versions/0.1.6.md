# Version 0.1.6

## Scripts → Skills Consolidation

The standalone `scripts/` directory has been retired. Each executable now lives next to the skill whose `*.md` describes how to use it, so the skill folder is fully self-contained.

### File Moves

- `scripts/update_jobs.ts` → split by command intent:
  - `skills/job_crawl/update_jobs.ts` — `add-job`, `list-events`, `list-entities` (agency/job ingestion path)
  - `skills/news_crawl/update_news.ts` — `add-news`, `list-events`, `list-entities` (two-entity news ingestion path)
- `scripts/cleanup_duplicates.ts` → `skills/news_crawl/cleanup_duplicates.ts` (entity-slug deduplication maintenance utility)
- `scripts/pipeline.py` → `skills/news_crawl/pipeline.py`
- `scripts/requirements.txt` → `skills/news_crawl/requirements.txt`

### Bug Fixes

- `pipeline.py`: removed a stray `resp.json()Str = resp.json()` line (line 60) that was invalid Python and would have crashed the LLM assessment branch at runtime. The single `result_text = resp.json()["choices"][0]["message"]["content"]` line is now correct.
- `pipeline.py`: adjusted the `load_prompts()` path to climb two directories instead of one, since the file now lives at `skills/news_crawl/pipeline.py` rather than `scripts/pipeline.py`. `config/prompts.yaml` is still loaded from the repo root.

### Skill Documentation Updates

- `skills/job_crawl/job_crawl.md`: every example now invokes `npx tsx skills/job_crawl/update_jobs.ts …`.
- `skills/news_crawl/news_crawl.md`: examples now call `npx tsx skills/news_crawl/update_news.ts …`, and a new **Tools** section enumerates the four colocated files. Added a **Maintenance — Purge duplicate entities** section documenting `cleanup_duplicates.ts`.

### CI / Workflow

- `.github/workflows/pipeline.yaml`: updated `pip install -r` and `python …` steps to reference `skills/news_crawl/requirements.txt` and `skills/news_crawl/pipeline.py`. (The workflow itself is still scheduled — the separate task of removing the backend probing action is not yet executed.)

### Deletions

- `scripts/` directory removed entirely.

### Verification

- `npx tsc --noEmit` across the whole project passes. TypeScript's glob-based `include: ["**/*.ts"]` picks up both relocated scripts without config changes.
