# Version 0.1.7

## Three-tab Graph, Inline Entity Edit, and Career Job Tracker

### New Tab: News (graph filter)

- Added a third top-bar tab ("News") in [src/app/page.tsx](src/app/page.tsx) alongside Money and Career.
- New `kind` query param on [`/api/graph`](src/app/api/graph/route.ts): `all` (default, Money view), `news` (events linking ≥2 entities), or `job` (single-entity events tied to an agency).
- `GraphCanvas` now accepts a `kind` prop and re-fetches when it changes.
- **Note:** The kind discriminator currently infers from entity count. A later version will replace this with an explicit `Event.category` column (tracked in `docs/TASK.md`).
- The multi-file upload affordance described in the original task is deferred — the user asked for it to be implemented later via an embedded OpenClaw conversation protocol. Added back to `docs/TASK.md` for future work.

### Career Tab: Hybrid Graph + Job Sidebar

- New component [src/components/CareerSidebar.tsx](src/components/CareerSidebar.tsx) — shown only when the Career tab is active. Lays out left of the main graph area.
- New API route [src/app/api/jobs/route.ts](src/app/api/jobs/route.ts) that returns job events filtered by:
  - `agency` (entity slug)
  - `q` (keyword match against title + description, case-insensitive)
  - `yoeMin`, `yoeMax` (years of experience, parsed from free-text title/description via a small regex)
- Sidebar provides debounced keyword search, agency dropdown (auto-populated from the unfiltered result set), and min/max YOE range inputs.
- Clicking an agency chip on a job card highlights the corresponding node in the graph via the existing `onNodeClick` handler.

### Inline Entity Edit in SidePanel

- [`/api/node`](src/app/api/node/route.ts) now supports `PATCH` with `{ homepage, jobPortal, description }`. URL fields are validated as `http`/`https` and empty strings are normalized to `null`.
- [SidePanel](src/components/SidePanel.tsx) node view gains an **Edit** button that swaps the read-only display for a form with description, homepage, and job portal inputs. Save / Cancel buttons; errors are surfaced inline.
- The optimistic local update keeps the panel in sync after a successful save without re-fetching the full node.

### Version & Docs

- `VERSION` bumped from `0.1.6` → `0.1.7`.
- `docs/TASK.md`: removed the four completed items, added two new follow-up items (deferred multi-file upload via OpenClaw, and `Event.category` migration).

### Verification

- `npx tsc --noEmit` passes across the project.
- Ran `next dev` at `localhost:3000` and hit each new API endpoint with `curl`:
  - `/api/graph?days=9999&kind=news` returns `{nodes:[],edges:[]}` against the current seed (expected — all seed events have only 1 entity).
  - `/api/graph?days=9999&kind=job` returns the three seeded Glasswing entities as nodes.
  - `/api/jobs` returns all 3 seed events as jobs.
  - `PATCH /api/node?id=anthropic` with a full payload succeeds and echoes the updated record.
- Browser-level UI verification was not run — no Playwright/screenshot tooling available in this environment. Recommend a manual smoke-test before pushing.
