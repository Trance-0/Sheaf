# Version 0.1.9

## Retire backend pipeline action, wire Vercel deploy

Sheaf is a single-user local-first intelligence board; the nightly GitHub Action that shelled into OpenRouter + Neon to probe new events was a liability (leaking API keys into CI, drifting from the skills-based ingestion model, and no longer matching the "user's agents add entries" workflow we settled on in 0.1.6). This release strips the backend probe from CI and prepares the repo for a frontend-only Vercel deploy with first-party Analytics and Speed Insights.

### Changes

- **Removed** [.github/workflows/pipeline.yaml](.github/workflows/pipeline.yaml) — the daily cron that ran `skills/news_crawl/pipeline.py` against OpenRouter + `DATABASE_URL`. Events are now added by the user running the ingestion skills (`skills/news_crawl/update_news.ts`, `skills/job_crawl/update_jobs.ts`, `skills/news_crawl/seed_glasswing.ts`) or by an agent invoking them on their behalf. `skills/news_crawl/pipeline.py` itself remains as a manual tool — only the CI wrapper is gone.
- **Installed** `@vercel/analytics@^1.x` and `@vercel/speed-insights@^1.x` (2 new packages, 0 vulnerabilities).
- **Wired** both components into [src/app/layout.tsx](src/app/layout.tsx): `<Analytics />` and `<SpeedInsights />` render after `{children}` inside `<body>` as per the Next.js App Router quickstart. No per-page changes needed — the components self-initialize in production builds and no-op in dev, so local development is unaffected.

### Deploy notes

- Next deploy on Vercel should Just Work: Next.js 16 App Router, no custom `vercel.json` required. The existing `DATABASE_URL` env var still needs to be set in the Vercel project settings for API routes (`/api/graph`, `/api/node`, `/api/edge`, `/api/jobs`) to reach Neon — make sure it points at the production Neon branch, not the `sample.test.env` dev branch.
- Analytics and Speed Insights are free on the Hobby tier and activate automatically once the app is deployed under a Vercel project. No env vars needed.
- The `GITHUB_SECRETS` previously holding `OPENROUTER_API_KEY` and `DATABASE_URL` for the CI job can be deleted — nothing in the repo references them after this version.

### Verification

- `npx tsc --noEmit` → clean (no type errors introduced by the new imports).
- `.github/workflows/` is now empty; the only CI surface remaining is whatever Vercel adds on connect.
