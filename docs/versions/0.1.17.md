# Version 0.1.17

## Intern / new-grad job seed from the active settings file

This round adds a hand-curated intern / new-grad job seeder that cross
references the active `sheaf-settings-*.json` file against a
hand-written roster of early-career programs and creates
`category='job'` events for each matching program.

### New: `intern_entry_seed.json`

**[skills/job_crawl/intern_entry_seed.json](skills/job_crawl/intern_entry_seed.json)**
is a hand-curated roster of 29 intern / new-grad programs across 21
tech entities (`alphabet`, `microsoft`, `apple`, `meta`, `amazon`,
`aws`, `nvidia`, `intel`, `amd`, `oracle`, `ibm`, `adobe`, `salesforce`,
`uber`, `figma`, `tiktok`, `bytedance`, `palantir`, `crowdstrike`,
`anthropic`, `openai`, `jpmorgan`, `tsmc`). Each program carries:

- `entityId` — the Sheaf entity slug to link the job Event to
- `title` — stable, used for event-level dedup
- `url` — canonical careers-page URL with a program-specific query param
  or path so article-URL dedup stays stable
- `postedAt` — approximate cycle-open date inside the last 12 months,
  so the default 1-year `DateRange` filter surfaces them
- `level` — `"intern"` or `"new-grad"`
- `description` — one-line context on what the program covers
- `relevantSkills` — tag array drawn from the `skillsKeywords`
  vocabulary (`Python`, `CUDA`, `PyTorch`, `Deep Learning`, etc.) —
  this is what the filter step uses

Programs include the well-known cycle names: Google STEP, Meta
University, Microsoft Explore / Aspire, Amazon Propel, Nvidia DL
Research Intern, Palantir PATH, Salesforce Futureforce, IBM Accelerate,
Anthropic Research Engineer Intern, OpenAI Residency.

### New: `seed_intern_entry_jobs.ts` driver script

**[skills/job_crawl/seed_intern_entry_jobs.ts](skills/job_crawl/seed_intern_entry_jobs.ts)**
is a standalone CLI that:

1. **Locates the active settings file.** Defaults to the
   lexicographically-greatest `sheaf-settings-*.json` in the repo root
   (dated filenames, so newest wins); `--settings=<path>` overrides.
2. **Loads `jobsConfig.skillsKeywords`** through a local copy of the
   `stripTrailingCommas` helper (same algorithm as
   `src/lib/useAppSettings.ts`, duplicated here so the CLI has no
   frontend dependencies and stays `tsx`-runnable).
3. **Filters programs** by skill intersection: a program is kept if any
   of its `relevantSkills` equals (case-insensitive, exact-token) any
   entry in the user's `skillsKeywords`. We deliberately don't use
   substring matching — `"Java"` must not match `"JavaScript"`.
4. **Scopes to entities with a `jobPortal`.** For each filtered
   program, the script looks up the target entity in the DB and skips
   it if the entity doesn't exist or has no `jobPortal` — explicitly
   requested by the task wording "for each tech entity with a
   `jobPortal`".
5. **Dedupes by article URL first, then by (entity, title) pair.** If
   the `Article.url` exists the program is skipped entirely; if a
   matching event title already lives on the target entity, the new
   article is attached to the existing event as an additional source
   (same pattern as the prior `update_jobs.ts add-job` handler).
6. **Otherwise creates** a new `Event` with `category: 'job'`, one
   `EventEntity` link (impact `5d=0.5`, `5w=1.0` — intentionally
   low-weight so intern postings don't dominate market-moving news),
   and one `Article` pointing at the canonical URL.

CLI flags:

- `--dry-run` — run the load + filter step and print what *would* be
  seeded, but do not touch the database. Useful for validating a new
  `skillsKeywords` list without the credits hit.
- `--settings=<path>` — point at a specific settings file instead of
  auto-discovering one.

### Execution against Neon

```
[settings] sheaf-settings-2026-04-11.json
[settings] 26 skillsKeywords loaded
[seed]     29 programs loaded
[filter]   29 / 29 programs match skillsKeywords
Summary:   29 created, 0 attached, 0 skipped (dup url),
           0 skipped (no portal), 0 skipped (no entity).
```

A second run immediately after produced `0 created, 0 attached,
29 skipped (dup url)` — the idempotency guard is correct.

`npx tsx skills/job_crawl/update_jobs.ts list-events` confirms every
program appears as a single-entity `category='job'` event in the 2025
Aug–Oct date range, so they'll show up inside the default 1-year
`DateRange` window with the v0.1.15 edge-alpha logic rendering them as
fresh.

### Not seeded (by design)

- **Public-sector agencies** (`dod`, `doe`, `nnsa`, `gsa`, `ftc`,
  `doj`, `ec`, etc.) — no `jobPortal` set; the task scopes this seed
  to "tech entities with a jobPortal".
- **Slack** — covered by the Salesforce Futureforce program, no
  separate seed entry needed.
- **Linux Foundation / OpenSSF** — nonprofits; no intern cycles.
