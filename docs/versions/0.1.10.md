# Version 0.1.10

## Graph usability pass

Round of fixes targeted at the complaints that came in after the Glasswing seed
landed: most of the graph felt missing, nodes jittered, drag didn't stick, the
News tab came up empty, edge clicks silently did nothing, and the sigma canvas
was still painting itself white under dark mode. All of these traced back to a
handful of root causes — the 30-day default filter hid almost every seeded
event, the ForceAtlas2 worker was never stopped so it fought every drag, and
the default `--sigma-background-color` CSS variable was never overridden in
dark mode.

### Graph & layout

- **ForceAtlas2 auto-stops after a 3-second settle window.** Before, the
  worker ran forever — nodes oscillated under gravity and drag updates were
  immediately re-overwritten by the next layout tick, so nothing would stick.
  [src/components/GraphCanvas.tsx](src/components/GraphCanvas.tsx) now combines
  layout + events into a single `SigmaController` that starts FA2, lets it
  settle for 3s, then calls `stop()`. `downNode` also calls `stop()`
  defensively so that clicking-to-drag before the timer fires still works.
- **Drag tracks globally.** Mousemove/mouseup listeners attach to
  `document` while a node is being dragged so the interaction doesn't break
  when the cursor leaves the canvas bounds.
- **Initial spread widened** from `Math.random() * 100` to
  `(Math.random() - 0.5) * 1000`, so FA2 starts from a spread cloud instead
  of a pile at the origin.
- **Remount on filter change.** The `SigmaContainer` now takes a
  `key={timeFilter}-${kind}-${graph.order}-${graph.size}` so every data reload
  gets a fresh layout worker + settle timer instead of inheriting whatever
  state the last graph was in.

### Dark mode canvas background

- [src/app/globals.css](src/app/globals.css) now overrides
  `--sigma-background-color` on `:root.dark` (sigma pins the default to `#fff`
  at `:root`, so the variable has to be re-set at equal or greater specificity
  to take effect) and bumps the `.sigma-dark` rule to
  `div.react-sigma.sigma-dark` so it beats sigma's own
  `div.react-sigma { background: var(--sigma-background-color); }` selector.

### Time filter → always-on scale bar

- **New component** [src/components/TimeScaleBar.tsx](src/components/TimeScaleBar.tsx):
  a glass-panel horizontal slider with stops at 1wk / 1mo / 3mo / 6mo / 1yr /
  2yr / 5yr / All. It displays the resulting absolute cutoff date ("Showing
  since Jan 7, 2025") so the filter is unambiguous no matter what "today" is,
  and each tick label is also a clickable shortcut.
- **[src/app/page.tsx](src/app/page.tsx)** replaces the hover dropdown with
  the scale bar pinned `top-20 left-1/2 -translate-x-1/2` (below the tab row
  so it doesn't collide on narrower screens).
- **Default changed from 30 days → 9999 ("All Time")**. 30 days hid almost
  every Glasswing-seed event on first load, which is what made "News entry
  not working" and "not all nodes are loaded" look like bugs. News and Money
  tabs both now populate immediately because the default filter no longer
  truncates the 2024–2026 event history.

### Recent-events drill-down in SidePanel

- **[src/app/api/node/route.ts](src/app/api/node/route.ts)** now includes
  `description`, `primaryArticleUrl`, `primaryArticleTitle`, and
  `primaryArticleProvider` on each recent event, pulled via a `take: 1`
  include of the newest article per event.
- **[src/components/SidePanel.tsx](src/components/SidePanel.tsx)** recent
  events list is no longer a flat row. First click on a row expands it
  inline with the short description, source provider, and a "click title to
  open" hint. Second click on the title while expanded opens
  `primaryArticleUrl` in a new tab via `window.open(..., "_blank",
  "noopener,noreferrer")`. If no article is attached the expansion still
  shows the description and a "no source attached" note.
- Separate `expandedRecent` state keeps this two-step interaction
  independent from the existing `expandedEvent` state used in the edge view.

### Why edge clicks now work

There was no regression in the `clickEdge` wiring itself — `enableEdgeEvents:
true` and the handler were both already in place. The reason "nothing
prompts" was that the 30-day default + the constantly-running FA2 meant (a)
almost no edges existed to click in the first place and (b) whatever few did
exist were being nudged around continuously. Fixing the filter default and
the layout settle window restores edge selection as a side effect.
