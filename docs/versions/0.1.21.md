# Version 0.1.21

## Browser-only database access: the Next.js server stops touching Postgres

Previous versions proxied every read and write through Next.js API routes
(`/api/graph`, `/api/node`, `/api/edge`, `/api/jobs`, `/api/migrate`).
Each route built a Prisma client from the user-supplied `DATABASE_URL`
header, queried Neon, serialized the result to JSON, and shipped it to
the browser. That worked until the deployed Prisma client went stale
relative to its own `schema.prisma` ‚Äî at which point *everything*
(including the in-browser migration page from 0.1.20) broke, because
Prisma couldn't initialize.

v0.1.21 removes the Next.js server from the data plane entirely. The
browser now talks to Neon directly over HTTP using
`@neondatabase/serverless`, so a broken Vercel build can't block data
access or schema recovery. The server only serves the static bundle.

### New client-side data layer

Everything used to round-trip through `/api/*`. Now each page-facing
fetch calls a typed client module that issues the SQL directly:

- **[src/lib/client/neon.ts](../../src/lib/client/neon.ts)** ‚Äî the only
  place that imports `@neondatabase/serverless`. Exports `createSql()`
  (returns a Neon query function with `fullResults: true` so callers
  get `rowCount` on DML), `withDriftDispatch()` (wraps any call so a
  schema-drift error re-dispatches the same `sheaf:backend-upgrade-required`
  event the old server used to emit), and `bigintToNumber()` for the
  `bigint` columns that come back from `COUNT(*)::bigint` summaries.
- **[src/lib/client/graphData.ts](../../src/lib/client/graphData.ts)** ‚Äî
  replaces `GET /api/graph`. Runs one flat `JOIN` between `Event`,
  `EventEntity`, and `Entity`, scoped to the date range and optional
  `kind` filter, then builds the same node/edge aggregation in JS that
  the old server did. Exports `GraphNode` / `GraphEdge` / `GraphPayload`
  types, which `GraphCanvas.tsx` now consumes directly.
- **[src/lib/client/nodeData.ts](../../src/lib/client/nodeData.ts)** ‚Äî
  replaces `GET /api/node` and `PATCH /api/node`. Uses three parallel
  queries (entity + aliases + latest snapshot + 30 most recent events
  with article counts) and reassembles the `NodeDetail` shape the side
  panel expects. `updateNode()` emits a single `UPDATE` with only the
  provided fields, matching the old `PATCH` contract.
- **[src/lib/client/edgeData.ts](../../src/lib/client/edgeData.ts)** ‚Äî
  replaces `GET /api/edge`. One self-join finds events where both the
  source and target entities participate, then two follow-up queries
  fetch the articles and impact scores for those events in bulk.
- **[src/lib/client/migrations.ts](../../src/lib/client/migrations.ts)** ‚Äî
  replaces `POST /api/migrate`. Runs the same idempotent SQL that the
  CLI uses (`ALTER ... ADD COLUMN IF NOT EXISTS`, backfill `UPDATE`s),
  produces the same `MigrationReport` shape, but talks to Neon directly.
- **[src/lib/client/backup.ts](../../src/lib/client/backup.ts)** (NEW) ‚Äî
  pulls every row from every Sheaf table via Neon HTTP and downloads a
  JSON snapshot. This is a **data-only** backup ‚Äî schema, indexes, and
  constraints are not captured. For a full structural backup the UI now
  tells users to create a Neon branch; the JSON dump is a supplemental
  convenience before running a migration.

### Updated consumers

- **[src/components/GraphCanvas.tsx](../../src/components/GraphCanvas.tsx)**
  now calls `fetchGraph({ databaseUrl, start, end, kind })` instead of
  `apiFetch("/api/graph?...")`. The inline `GraphNode` / `GraphEdge`
  interfaces are deleted in favour of the ones exported from
  `graphData.ts`.
- **[src/components/SidePanel.tsx](../../src/components/SidePanel.tsx)**
  calls `fetchNodeDetail`, `updateNode`, and `fetchEdge` directly. The
  inline `NodeData` / `EventData` interfaces become type re-exports
  from the client modules.
- **[src/components/BackendUpgradePrompt.tsx](../../src/components/BackendUpgradePrompt.tsx)**
  collapses the two old branches (`schema-drift` / `prisma-client-stale`)
  into one flow. The plan preview is populated synchronously from
  `listBrowserMigrations()` instead of a GET request, and a new
  "Download data backup" section runs `captureBackup` + `downloadBackup`
  before the user acknowledges the backup and commits the migration.

### Retired server-side code

- **Deleted:** `src/app/api/graph/`, `src/app/api/node/`,
  `src/app/api/edge/`, `src/app/api/jobs/`, `src/app/api/migrate/`,
  `src/lib/server/`.
- **Simplified:** `src/lib/apiFetch.ts` down to just the
  `BACKEND_UPGRADE_EVENT` constant and `BackendUpgradeInfo` type. The
  old fetch wrapper that inspected 5xx responses is gone ‚Äî no server
  code path produces those signals anymore, so the wrapper is dead code.
- **CLI-only migrations:** `prisma/migrate_auto.ts` now composes the
  `MIGRATIONS` list inline (importing `runSnapshotFinancialsMigration`
  and `runEventCategoryMigration` directly from the sibling
  `prisma/migrate_*.ts` files), since `src/lib/server/migrations.ts`
  is gone. The CLI path stays useful as the maintainer-facing fallback
  for non-Neon Postgres, where browser HTTP tunnelling is not available.

### Security and deployment implications

- The `DATABASE_URL` is still browser-held, as it has been since 0.1.13.
  Nothing about the trust boundary changes ‚Äî the browser was already
  forwarding the URL to the app's own backend, and the backend was
  already querying Neon with it. Now the intermediate hop is gone.
- `@neondatabase/serverless` uses HTTPS to Neon's query endpoint, so
  there is no new CORS setup or websocket tunnelling required ‚Äî the
  driver works in the browser out of the box.
- **Non-Neon Postgres is no longer supported from the browser.** The
  HTTP driver only talks to Neon's proxy. If a user configures a
  different Postgres URL they get a clear SQL-side error; the CLI
  driver (`npx tsx prisma/migrate_auto.ts`) remains available as an
  escape hatch.
- The only remaining API route is `/api/settings`, which still returns
  `410 Gone` from 0.1.13.

### Verification

- `npm run lint` ‚Äî passes.
- `npx tsc --noEmit` ‚Äî passes after clearing `.next/`.
- `npx next build` ‚Äî passes. Build output shows only `/api/settings`
  as a dynamic route; everything else is static.
- End-to-end browser testing against a live Neon database is pending
  and should be done manually after this lands: load the graph, open
  the side panel for a node and for an edge, edit an entity, and (on a
  deliberately-drifted dev database) trigger the backup + migration
  flow.
