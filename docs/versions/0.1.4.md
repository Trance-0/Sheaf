# Version 0.1.4

## Major UI Overhaul & Interactive Graph Improvements

### TailwindCSS Migration
- Replaced all Vanilla CSS with TailwindCSS v3 throughout the project
- Updated `AGENTS.md` to mandate TailwindCSS for all future UI work
- Rewrote `globals.css` to use `@tailwind base/components/utilities` with a minimal set of CSS custom properties for theming
- Configured `tailwind.config.js` with `darkMode: 'class'` strategy and correct content paths
- Refactored `page.tsx`, `SidePanel.tsx`, `SettingsPanel.tsx` to use Tailwind primitives exclusively

### Edge Interactivity
- Enabled `enableEdgeEvents: true` in Sigma settings
- Wired `clickEdge` event handler in `GraphCanvas.tsx` → `onEdgeClick` prop passed up to `page.tsx`
- `SidePanel.tsx` now accepts both `selectedNode` and `selectedEdge` props, rendering event cluster detail when an edge is clicked

### Time Filter
- Added a 5D / 5W / 5M / All Time dropdown in the top bar
- `timeFilter` state passed down to `GraphCanvas` as a prop
- `/api/graph` now accepts a `?days=N` query param and filters events via `prisma.event.findMany({ where: { date: { gte: cutoffDate } } })`

### Label Contrast Fix
- `labelColor` in Sigma settings dynamically set based on body theme class: white text on dark, dark text on light

### Database Deduplication
- Ran `scripts/cleanup_duplicates.ts` — confirmed no duplicates presently
- Script normalizes entity names to slugs for comparison; re-runnable at any time
- `/api/graph` now normalizes all entity IDs to lowercase before building graph to prevent visual duplicates from casing differences
