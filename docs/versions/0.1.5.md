# Version 0.1.5

## Event-Centric Edge Architecture

### Data Model Changes
- Added `homepage` and `jobPortal` fields to `Entity` model in Prisma schema
- Schema pushed to Neon Postgres

### API Overhaul
- **`/api/graph`**: Edges are now collapsed by entity-pair. Each visual edge carries a summed `weight` (from impact scores) and an `eventCount`. Multiple events between the same pair produce ONE edge with thickness proportional to total weight.
- **`/api/edge`** (NEW): Returns the full event→article drill-down for a given source-target entity pair.
- **`/api/node`** (NEW): Returns entity details (homepage, job portal, description, recent events). No articles on nodes.

### UI Changes
- **SidePanel — Node view**: Shows entity type, description, homepage link, job portal link, and recent events. Removed "Linked Articles" section from node view.
- **SidePanel — Edge view**: Shows a collapsible list of events. Clicking an event expands it to show the event description, impact scores per entity, and linked article sources.
- **GraphCanvas**: Edge click now extracts the source-target pair from the graph and passes it upstream. Edge visual size scales with summed weight.

### CLI & Skills
- **`scripts/update_jobs.ts`**: Full rewrite with dedup logic. New `add-news` command for multi-entity events. Probes DB by keyword before inserting — if a matching event already exists, attaches the article as a reference instead of creating a duplicate event.
- **`skills/job_crawl/job_crawl.md`**: Rewritten with dedup-first protocol instructions.
- **`skills/news_crawl/news_crawl.md`** (NEW): Created with event-centric data model instructions and dedup protocol.
